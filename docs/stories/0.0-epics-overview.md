# TillLess Phase 1 Epic Overview

## Epic 1 – Data Backbone Foundations
Establish reliable, timely retailer data via the Canonical Product Registry, Temporalite-scheduled Playwright scrapers, pg-boss ingestion, and monitoring. Stories 1.1–1.4 cover Checkers bootstrapping, additional retailers, and dashboards.

## Epic 2 – Optimisation Engine & API
Design and implement the NestJS optimisation service that consumes CPR data via Prisma, applies promotion and substitution logic, factors travel costs, and exposes REST/GraphQL endpoints. Stories will include algorithm design, promo evaluation, travel heuristics, and API integration tests.

## Epic 3 – Shopping List Experience (Frontend)
Deliver the Next.js UI for list entry/import, preference management (loyalty toggles, effort tolerance), and optimisation results with savings explanations. Stories include list management UI, preferences UI, results dashboard, PDF/email export.

## Epic 4 – Account & Auth Enablement
Integrate BetterAuth for user registration, login, and session management; manage loyalty card data, profiles, and feature gating. Stories cover BetterAuth configuration, NestJS guards, account management UI, and secrets handling.

## Epic 5 – Feedback Loop & Analytics
Implement receipt reconciliation and savings tracking to capture actual spend, adjust mappings, and surface insights. Stories include receipt upload pipeline, reconciliation UI, accuracy metrics, and analyst review tooling.

## Epic 6 – Ops & Reliability Enhancements
Harden observability, alerting, and runbooks beyond ingestion dashboards, including deployment automation, secret management, and rollbacks. Stories cover infrastructure automation, on-call handbook, backup/restore, and performance tuning.

*Future Epics*
- **Epic 7 – Advanced Features (Phase 1.5+)**: Receipt OCR, Food Lover's expansion, multi-store routing pilots, recommendation engines (scoped for future phases).
